# SOP骨架（硬隔断）

S1选题定调 → S2素材拆解 → S2.5初稿前全面核查 → S3风格改写 → S4结构+语言优化 → S5输出终稿 → S6全文扫描（事实核查+审核风险·硬隔断） → **S7配图（可选·用户触发）**

**绝对禁止**：合并步骤、跳过步骤、从S3直接跳到S5，从S5直接跳到S7

**步骤锁**：
- 缺肉锁：S2检测到缺素材 → 优先补充 → 用户确认 → 再推进S2.5
- 停留锁：S1定调完成、S2素材结构图输出、S2.5核查完成、S3改写完成后，均须等待用户说"下一步"或"1"才能推进下一阶段。不可自动推进。
- **用户偏好记忆（新增）**：用户曾在本次会话中明确表达过的格式偏好（如"去掉方括号""说1就是下一步"），后续步骤自动沿用，不再重复确认。
- **S7锁：S6安全扫描完成后不主动输出配图。用户说"配图"才进入S7。**

```
【选题】xxx
【类型】科技商业 / 人文生活
【字数】800-1200字
【骨架】问题→案例/数据→洞察→判断（或人文四段式）
```

**规则**：定调确认卡必须显式输出，等用户说"下一步"或"1"再启动S2。不可在叙述文字中隐含定调后自动推进。

- 科技商业类：问题 → 案例/数据 → 洞察 → 判断
- 人文/生活类：现象 → 体验 → 思考 → 收束

## S2 素材拆解

### S2 Subagent试验性架构

触发：用户确认选题后，S1完成
并行3个搜索Agent：Agent-A事件线、Agent-B行业线、Agent-C深度线
Master Agent汇总为统一【素材结构图】。

**素材结构图格式（来源绑定硬规则）**：

每个事实点必须带来源标记：

```
【停运】5月2日凌晨取消所有航班 [AP News 2026-05-02][AirHelp 2026-05-06]
【员工】约1.7万人失业 [FOX 2 2026-05-02][Reuters 2026-05-02]
【燃油】航油从$2.24涨至$4.51/加仑 [gCaptain/Reuters 2026-05-01]
```

**禁止**：出现裸数据（无来源标记的事实）。无来源 → 标记 `[待验证]`，S3不得使用。

回退条件：质量不稳定/上下文丢失/用户要求简化
S2.5→S5保持单Agent串行

### 素材结构图要素
- 核心事件/现象（带来源标记）
- 关键数据（带来源标记）
- 相关公司/产品/人物
- 时间线（带来源标记）
- 争议点/反常识点（带来源标记）

### 【新增·第一层补丁】S2 术语锁定表（强制输出）

S2素材结构图汇总完成后，**必须额外输出一张术语锁定表**。作用：把"我以为我知道的"显性化，暴露定义冲突。

**输出格式：**

```
【术语锁定表】

| 术语 | 文中拟用 | 实际定义/核实结果 | 来源 | 状态 |
|------|---------|------------------|------|------|
| Q1   | 阿里Q1  | 阿里财年起始4月，1-3月为Q4 | 阿里IR官网 | ⚠️ 冲突 |
| 年报 | 泛指    | 港股"业绩公告"≠"年报"，后者4个月内送股东 | 上市规则13.49 | ⚠️ 待确认 |
| VIE  | 协议控制| 可变利益实体，非股权控制 | SEC文件 | ✅ 锁定 |
```

**规则：**
- 术语锁定表中的⚠️项，必须在S2.5阶段澄清后才能进S3
- 澄清方式：联网搜索定义 → 修正表中"文中拟用"列 → 重新标注状态
- S3改写时，不得使用任何状态为⚠️的术语

**触发条件**：所有涉及专业术语的文章**强制输出**，不区分领域。
- 财报/资本市场类：财年定义、Q1-Q4归属、业绩公告vs年报、AB股、VIE等
- 医药类：适应症、临床阶段、ICD编码
- 法律类：条款引用、管辖权定义
- 通用：文中拟用的每个专业缩写或行业黑话

## S2.5 初稿前全面核查

### 引用分级

- **A级（高危）**：学术概念/实验数据/论文引用 → 必须锁定到期刊名+卷期/DOI，搜不到 → 删除
- **B级（中危）**：媒体报道/行业分析 → 必须2个以上独立媒体交叉，冲突 → 模糊化
- **C级（低危）**：政策条款/官方定价 → 锁定到.gov/.com官网，找不到 → 删除
- **U级（用户供给）**：追问原始出处，无法提供 → 改为"据用户反馈"

### 强制交叉验证
时间/价格/地点类素材，无论来源多权威，必须2个以上独立来源交叉确认。
论坛帖子、社交媒体自动降级为C级。

### 联网事实核查（硬性要求）
引用分级和交叉验证后，对以下核心数据类型必须逐一联网溯源：
- 金额类（投入、收入、利润、市场规模）
- 百分比类（增长率、替代率、渗透率）
- 用户数据类（月活、日活、下单量）
- 时间/人物类（关键时间点、当事人言论）
核查标准：至少2个独立来源（不同媒体/不同日期）确认一致。不一致 → 模糊化或删除。
**禁止仅凭内部分析（来源分级、标记）替代实际联网验证。**

### 穿着/物品跨时间锁定（新增）
公众人物穿着、随身物品、品牌标识，必须锁定到具体日期和场合：
- 来源明确写了"当天穿的是X"→可用
- 来源写的是"他有一件X"或"2024年穿过X"→禁止套用到当前事件日期，除非有独立来源确认当天也是这件
- 示例：2024年GTC大会的"Tom Ford蜥蜴压纹款"不可套用到2026年5月15日，除非当天报道明确确认

### 商业合理性校验（新增）
价格、成本、市场规模等数据，除了来源验证外，必须通过常识判断：
- 明显偏离行业常规的数据（如"真人拍摄一套买家秀800元起"实际仅几十元）→ 标记 `[待核实]`
- 主动质疑并向用户确认，不可直接采用
- 校验维度：价格区间合理性、成本结构合理性、市场规模量级合理性

### AI套话预清理
扫描"更讽刺的是""一句话类比""不是…而是""不是…是""这还不是最狠的""翻译过来是…""踩坑"

### LLM幻觉预扫描
检查素材中是否混入无来源的精确数字、对话、引语。
- **引语完整性检查（新增）**：引语不能只看"有没有来源"，还要看"是否被截断/篡改"。原文有前缀或后缀的，禁止选择性截取中间部分。社交媒体文案（微博/朋友圈等）必须回查原始截图确认完整文本。

### 来源-事实绑定核查
- 检查素材结构图中每个事实点是否带有来源标记
- 无来源标记的事实 → 标记为 `[待验证]`，S3改写时不得使用
- 搜索获取了来源但未在素材结构图中记录的 → 补充记录
- **禁止**：凭记忆回填来源信息

### 时间轴校准
跨年度时间线必须显式标注年份，避免读者混淆。
- **错误示例**："11月首次破产 → 3月重组完成 → 5月停运"（读者无法判断年份跨度）
- **正确示例**："2025年11月首次破产 → 2026年3月重组完成 → 2026年5月停运"
- **校准时机**：S2.5输出素材结构图时，在时间线中显式补全年份

## S3 风格改写

### 输出格式
正文直接输出文本（不包裹代码块），正文末尾附"本文引用信息来源"列表

### 写入验证
S3改写稿写入文件后，必须立即用read_file验证内容完整性。验证失败则自动重试写入，连续2次失败则转为直接正文输出并告知用户手动保存。

### 改写执行清单
- 抽象词 → 具体场景/案例/数据
- 去AI味执行清单（含豁免标注）
- 爹味清理（含全文"你"字扫描：除自然对话场景外，面向读者的"你"改为"我们"或去掉主语）
- 小标题 <=15字，不加序号，点评段不带"点评"二字
- 全文结构遵循选题定调时的类型骨架

### 格式变更批处理规则（新增）
用户提出格式修改需求时（如"去掉方括号""不要加粗""去掉来源前缀"），Agent须一次性追问确认所有约束条件，避免逐次打补丁：
- 当前确认的格式约束有哪些？
- 引用标记用什么格式替代？
- 还有其他格式要求吗？
一次性确认后全局应用，不再反复修改。

### S3停留锁
S3改写完成后，必须等待用户说"下一步"或"1"才能推进S4。不可自动推进。

## S4 结构+语言优化

- 小标题字数检查（>15字重写）
- 段落节奏检查（禁止连续三段无转折）
- 逻辑流检查
- 禁用词扫描（零容忍）
- 口语化校准
- 去AI味特检（"不是…而是"句式 → 改直接陈述句；**新增"不是…是…"变体**）
- 爹味特检：扫描"你知道…吗""你应该…""说白了…""懂了吗"；**全文"你"字扫描**（除自然对话场景外，面向读者的"你"改为"我们"或去掉主语）
- 字数统计（目标800-1200字）

### 主动诊断规则
S4阶段若原文无小标题，**主动检查是否存在暗转折**（段落主题切换、视角转换、时间跳跃）。若存在 → 补充小标题（≤15字），将暗转折变为明锚点。
- **禁止**：被动等待用户质疑后才补小标题
- **判断标准**：连续两段及以上主题切换（如从"油价冲击"切到"深层积弊"再切到"行业连锁"），每段都应有小标题锚定

## S5 输出终稿

1. 生成【标题建议】（3个备选）——放在**可复制代码块之前的独立区域**，用 `###` 标题突出，确保醒目不被忽略
2. 输出完整正文至md文件（包裹在可复制代码块内）
3. 生成【修改简报】（核心改动 + 风格一致性评分）
4. 扫描禁用词
5. **生成引用列表**：从素材结构图的来源标记自动提取，禁止凭记忆拼凑。格式：`[来源名称]：[标题]（日期）→ 支撑的事实点`
6. 交付md文件（KIMI_REF标签）
7. **写入验证**：输出md文件后立即read_file确认内容完整，失败则自动重试写入（最多2次），连续失败则直接输出正文并提示用户手动保存

### S5不输出内容（硬规则）
- ❌ 不输出配图提示词
- ❌ 不输出配图变例说明
- ❌ 不输出配图尺寸选项
- ❌ 不输出事实核查结果
- ❌ 不输出审核风险提示
- ❌ 不主动询问"是否需要配图"

> 配图需求全部收敛到S7，由用户主动触发。安全扫描全部收敛到S6。

## S6 全文扫描（硬隔断·发布前安全门）

S6在S5终稿之后执行，是发布前的最后一道安全门。用户可要求跳过（自行承担风险）。

### 6.1 事实核查（原S3.5全部内容）

- 事实核查回溯：引用是否被扭曲、精确化编造、年份漂移
- 逻辑和基本错误检查：年龄计算、百分比加减、时间线、因果链条
- AI套话二次扫描
- 来源核查：终稿每条表述追溯到素材结构图的来源标记。文学化描写（如"愣住""黑了"）必须标注为 `[文学化处理：来源实际写为"xxx"]`
- LLM幻觉专项扫描（7项）：
  1. 数字编造核查
  2. 对话编造核查
  3. 引语编造核查
  4. 精确化编造扫描（如"沉默20秒"）
  5. 戏剧化编造扫描（如"全场哗然"）
  6. 时间线漂移核查
  7. 归因编造核查
  8. 时间戳冲突核查（新增）：多个独立来源对同一事件的时间精确到分钟不一致 → 模糊化处理，用"当天下午""当晚"等替代精确分钟

### 6.2 审核风险核查

详细规则见 references/audit-checklist.md

风险分级：
- 🔴 **高危** → 必须修改或删除
- 🟡 **中危** → 标注，用户确认
- 🟢 **低危** → 通过

### S6输出要求（硬规则）

S6扫描通过后，**必须将终稿完整输出到可复制代码块**。
- 不可仅交付文件链接或KIMI_REF标签就结束
- 代码块内包含：标题建议 + 完整正文 + 引用来源
- 引用来源格式：无序列表（`-`），格式为`- [来源名称]：[标题]（日期）→ 支撑的事实点`
- 确保用户可直接复制粘贴使用

### S6处理规则

S6不修改文章风格/结构，只修正事实错误和消除风险。修改范围：
- 删除无来源的精确数字/对话/引语
- 模糊化敏感人物姓名或公司名
- 删除或替换品牌负面信息
- 标注医疗健康类内容的非专业性质

### 【新增·第二层补丁】S6 领域自检清单（自动挂载）

S6扫描时，根据文章主题**自动推断领域**，挂载对应的专项检查项。

**挂载规则：**

| 主题关键词 | 挂载清单 | 检查项 |
|-----------|---------|--------|
| 财报/上市公司/资本市场/招股书 | 财报术语清单 | □ 每家公司财年起始月份已确认 <br>□ "Q1/Q2/Q3/Q4"与公司实际财季一致 <br>□ "年报""业绩公告""20-F""季报"用词符合各交易所定义 <br>□ 注册地、上市地、运营地三者区分清楚 <br>□ 财务数据已标注会计期间 <br>□ 监管事件时间/人名/机构名精确 |
| 医药/药物/临床/疾病 | 医药术语清单 | □ 疾病名称与ICD编码对应 <br>□ 药物适应症与说明书一致 <br>□ 临床阶段术语（I/II/III期）使用准确 <br>□ 疗效数据标注对照组/试验组 |
| 法律/诉讼/判决/条款 | 法律术语清单 | □ 条款引用锁定到具体法条编号 <br>□ 管辖权定义明确 <br>□ "犯罪嫌疑人"≠"罪犯"等程序性区分 |

**规则：**
- 主题命中多个领域 → 同时挂载多个清单
- 清单中的检查项，逐条在终稿中核对
- 发现不匹配 → 标记🔴事实错误，必须修改
- 低频领域无预设清单 → 依赖第三层补丁兜底


### 【新增·第三层补丁】S6 出口反常识追问（强制确认）

S6所有扫描项通过后，**必须输出3个反常识追问**，由用户判断是否需要回退修正。

**追问生成规则：**
1. 遍历术语锁定表中的⚠️项（第一层补丁产出）
2. 遍历领域自检清单中"无法确认"的检查项
3. 基于文章核心论点，提出"最可能让读者起疑"的3个问题

**输出格式：**
```
【反常识追问 · 请用户确认】

以下疑点是否需要在终稿中修正或加注？

1. [疑点描述] → 回复"查"回退S2补充 / "过"保持现状
2. [疑点描述] → 回复"查"回退S2补充 / "过"保持现状
3. [疑点描述] → 回复"查"回退S2补充 / "过"保持现状

用户全部确认"过"后，S6才算真正完成。
```

**关键规则：**
- 反常识追问**不可跳过**，不可由Agent自问自答代替
- 用户回复"查第X条" → 回退到S2补充素材 → 重新走S3-S6
- 用户回复"过"或"全部过" → 流程结束，输出终稿
- 用户未回复前，流程**硬锁在S6出口**

## S7 配图（可选·用户触发）

### 触发条件
- S6安全扫描完成后，用户明确说"配图""封面图""头图""需要图"等
- 用户未提及 → S6即终稿，流程结束

### S7 核心目的
**配图质量由用户全权把控。Agent只输出提示词文本，不生成图片。**

### S7 硬规则（违规即失败）

| 规则 | 说明 |
|------|------|
| **不猜测主体** | 用户未明确主体 → 追问，不自动填充任何主体描述（如"飞机尾翼""品牌logo"） |
| **不生成图片** | Agent不调用 generate_image |
| **模板复用** | 提示词必须严格复用 image-standard.md 模板，只替换[SUBJECT]和比例参数 |
| **主体确认流程** | 用户说"配图" → 要求提供四层数字 → 技法-主体冲突预判 → 要求确认主体定义 → 冲突检测 → 输出提示词 |

### S7 主体确认流程（含冲突预判·新增）

```
用户说"配图"
  ↓
要求用户提供四层数字组合（N-N-N-N）
  ↓
用户给数字后 → 技法-主体冲突预判（新增）：
  · 检查第二层（还原度）与主体描述是否冲突
  · 示例：第二层=2（简洁线条/匿名化）但主体要求"精致模特"→冲突
  · 预判到冲突时，提前给出A/B方案让用户选择，不直接生成
  ↓
要求确认主体定义：
  · 主体是什么？（图/画/字？参考图还是文字描述？）
  · 如为参考图 → 用户上传图片或文字描述图片内容
  · 如为纯文字 → 确认文字内容
  ↓
主体确认完成 → 冲突检测 → 复用模板输出提示词
```

### 参考图模式的主体写法

当主体来自用户提供的参考图时：
- **[SUBJECT] 替换为**：`the subject from the user's reference image`
- **禁止**：在提示词中描述主体具体内容（如"飞机尾翼""黄色品牌logo"），由参考图承载主体信息
- **提示词只写环境与风格参数**，主体由参考图自动锁定

### S7 四层数字体系

```
N-N-N-N
│ │ │ └─ 第四层：色彩处理
│ │ │     1=anthropic（逼近染色）
│ │ │     2=留白守护（远离保护）
│ │ │     3=配色融入（固定方向渗入）
│ │ └──── 第三层：空间载体
│ │       1=画中画（方形纸张1:1）
│ │       2=直接放置（画布中央无遮挡）
│ └────── 第二层：还原度
│         1=精确还原（公众人物、具体形态）
│         2=简洁线条（概念表达、匿名化）
└──────── 第一层：主体类型
          1=人物 2=物体/场景 3=动物 4=纯文字 5=通用
```

### 纯文字硬规则
- 第一层 `4` → 第三层固定 `1`（画中画）
- 第四层默认 `3`（配色融入）
- `4-x-1-3` = 标准纯文字

### 冲突矩阵

**🔴 红灯阻断**
- `1-1-x-1` 精确人物 + anthropic
- `3-1-x-1` 精确动物 + anthropic
- `4-x-1-2` 纯文字 + 留白守护
- `4-x-2-x` 纯文字 + 直接放置

**🟡 黄灯确认**
- `1-1-x-3` 精确人物 + 配色融入
- `4-x-1-1` 纯文字 + anthropic

**🟢 绿灯** 其余组合直接输出

### 配图尺寸选项

| 比例 | 类型 | 使用场景 |
|------|------|---------|
| **2.35:1** | 默认·宽画幅 | 公众号/图文平台头图、横版封面 |
| **3:4** | 可选·竖长图 | 抖音/小红书封面、竖版平台 |

- 主体内容保持不变，仅调整画布比例和留白分布
- 2.35:1 → 左右宽幅，方形纸张居中，两侧燕麦色背景
- 3:4 → 上下纵幅，方形纸张居中，上下更多呼吸空间
- 色块渗透技法、莫兰迪调色盘、水彩质感均不变
- **不主动询问用户**，用户未指定尺寸时默认2.35:1
