# Publisher v5.5 - 观者范式创作流程

## 阶段总览

S1选题定调 - S2素材拆解 - S2.5初稿前全面核查 - S3风格改写 - S4结构+语言优化 - S5输出终稿 - S6全文扫描（事实核查+审核风险·硬隔断） - **S7配图（可选·用户触发）**

**硬隔断规则**：绝对禁止合并步骤、跳过步骤、从S3直接跳到S5，从S5直接跳到S7。

**引用文件**：
- 完整流程与各阶段详细规则：见 `references/workflow-s1-s7.md`
- 配图标准与变例系统：见 `references/image-standard.md`

## S1 选题定调

**输出格式（定调确认卡）**：

```
【选题】xxx
【类型】科技商业 / 人文生活
【字数】800-1200字
【骨架】问题->案例/数据->洞察->判断（或人文四段式）
【核心论点】一句话概括
```

**规则**：定调确认卡必须显式输出，等用户说"下一步"或"1"再启动S2。不可在叙述文字中隐含定调后自动推进。

文章类型决定骨架：
- **科技商业**：问题 -> 案例/数据 -> 洞察 -> 判断
- **人文/生活**：现象 -> 体验 -> 思考 -> 收束

---

## S2 素材拆解

### 并行搜索（Subagent模式）

S1完成后，启动3个并行搜索Agent：
- Agent-A（事件线）：核心事件、时间点、当事人动作
- Agent-B（行业线）：行业数据、市场规模、政策
- Agent-C（深度线）：论文原理、历史案例、反常识角度

Master Agent汇总为【素材结构图】。

**素材结构图格式（来源绑定硬规则）**：

```
【停运】5月2日凌晨取消所有航班 [AP News 2026-05-02][AirHelp 2026-05-06]
【员工】约1.7万人失业 [FOX 2 2026-05-02][Reuters 2026-05-02]
【燃油】航油从$2.24涨至$4.51/加仑 [gCaptain/Reuters 2026-05-01]
```

**禁止**：素材结构图中出现裸数据（无来源标记的事实）。

### 第一层补丁：术语锁定表（强制输出）

S2素材结构图汇总完成后，**必须额外输出一张术语锁定表**。作用：把"我以为我知道的"显性化，暴露定义冲突。

**输出格式**：

```
【术语锁定表】

| 术语 | 文中拟用 | 实际定义/核实结果 | 来源 | 状态 |
|------|---------|------------------|------|------|
| Q1   | 阿里Q1  | 阿里财年起始4月，1-3月为Q4 | 阿里IR官网 | ⚠️ 冲突 |
| 年报 | 泛指    | 港股"业绩公告"≠"年报" | 上市规则13.49 | ⚠️ 待确认 |
| VIE  | 协议控制| 可变利益实体 | SEC文件 | ✅ 锁定 |
```

**规则**：
- 术语锁定表中的⚠️项，必须在S2.5阶段澄清后才能进S3
- S3改写时，不得使用任何状态为⚠️的术语
- 触发条件：所有涉及专业术语的文章**强制输出**

**预设领域**：
- 财报/资本市场：财年定义、Q1-Q4归属、业绩公告vs年报、AB股、VIE
- 医药：适应症、临床阶段、ICD编码
- 法律：条款引用、管辖权定义

### 缺肉锁

S2检测到关键素材缺失 -> 优先补充素材 -> 用户确认 -> 再推进S2.5。不可跳过。

---

## S2.5 初稿前全面核查

执行以下核查：
1. **引用分级**（A学术/B媒体/C政策/U用户供给），按对应级别锁定出处
2. **强制交叉验证**：时间/价格/地点类素材必须2个以上独立来源交叉确认
3. **联网事实核查**：引用分级后，对核心数据逐一联网溯源
4. **商业合理性校验**：价格、成本、市场规模等数据必须通过常识判断
5. **AI套话预清理**：扫描并标记待删除句式
6. **LLM幻觉预扫描**：检查无来源的精确数字、对话、引语
7. **穿着/物品跨时间锁定**：公众人物穿着必须锁定到具体日期和场合
8. **时间轴校准**：跨年度时间线必须显式标注年份
9. **术语锁定表澄清**：所有⚠️项必须解决后才能进S3

详细规则见 `references/workflow-s1-s7.md`

## S3 风格改写

### 核心动作
- 抽象词 -> 具体场景/案例/数据
- 去AI味（默认删除"不是…而是"等句式，豁免有幽默感的变体）
- 爹味清理
- 小标题 ≤15字，不加序号

### 格式
正文直接输出文本（不包裹代码块），末尾附"本文引用信息来源"列表。

### 引用格式偏好（默认简化版）
```
- 雪球 Richard2858：腾讯、阿里巴巴2026年Q1财报日历（2026-05-11）
- 汉坤律所：《主要证券交易所上市公司手册》（2024-06-11）
```

**默认去掉箭头及事实支撑描述**。如用户需要详细格式，再追问确认。

### 停留锁
S3改写完成后，**必须等待用户说"下一步"或"1"才能推进S4**。不可自动推进。

---

## S4 结构+语言优化

### 执行内容
1. 小标题字数（>15字重写）
2. 段落节奏（禁止连续三段无转折）
3. 逻辑流检查
4. 禁用词扫描（零容忍）
5. 口语化校准
6. 去AI味特检（"不是…而是"->直接陈述句）
7. 爹味特检
8. 字数统计（800-1200字）

### 主动诊断规则
S4阶段若原文无小标题，**主动检查是否存在暗转折**。若存在 -> 补充小标题（≤15字）。

---

## S5 输出终稿

按顺序执行：
1. 生成【标题建议】（3个备选）——放在代码块之前的独立区域
2. 输出完整正文至md文件（包裹在可复制代码块内）
3. 生成【修改简报】（核心改动 + 风格一致性评分）
4. 扫描禁用词
5. 生成引用列表：从素材结构图的来源标记自动提取
6. 交付md文件（附KIMI_REF标签）
7. 写入验证：输出md文件后立即read_file确认内容完整

### S5不输出内容（硬规则）
- ❌ 不输出配图提示词
- ❌ 不输出配图变例说明
- ❌ 不输出事实核查结果
- ❌ 不输出审核风险提示
- ❌ 不主动询问"是否需要配图"

---

## S6 全文扫描（事实核查+审核风险·硬隔断）

### 定位
发布前安全门。S5终稿交付后，必须通过全文扫描才能发布或进入S7。

### 6.1 事实核查
- 事实核查回溯、逻辑检查、AI套话二次扫描
- 来源核查：终稿每条表述追溯到素材结构图的来源标记
- LLM幻觉专项扫描（8项）：数字/对话/引语/精确化/戏剧化/时间线漂移/归因/时间戳冲突

### 6.2 审核风险核查
- 🔴 **高危** -> 修改或删除
- 🟡 **中危** -> 标注，用户确认
- 🟢 **低危** -> 通过

### 第二层补丁：领域自检清单（自动挂载）

S6扫描时，根据文章主题**自动推断领域**，挂载专项检查项。

| 主题关键词 | 检查项 |
|-----------|--------|
| 财报/上市公司/资本市场 | □ 每家公司财年起始月份已确认 <br>□ "Q1-Q4"与公司实际财季一致 <br>□ "年报""业绩公告"用词符合各交易所定义 <br>□ 注册地、上市地、运营地三者区分清楚 |
| 医药/药物/临床 | □ 疾病名称与ICD编码对应 <br>□ 药物适应症与说明书一致 <br>□ 临床阶段术语使用准确 |
| 法律/诉讼/判决 | □ 条款引用锁定到具体法条编号 <br>□ 管辖权定义明确 |

**规则**：发现不匹配 -> 标记🔴事实错误，必须修改。

### 第三层补丁：反常识追问（强制确认）

S6所有扫描项通过后，**必须输出3个反常识追问**。

**追问来源**：
1. 术语锁定表中的⚠️项
2. 领域自检清单中"无法确认"的项
3. 文章核心论点中最可能让读者起疑的地方

**规则**：
- 反常识追问**不可跳过**，不可由Agent自问自答代替
- 用户回复"查第X条" -> 回退S2补充素材 -> 重新走S3-S6
- 用户回复"过"或"全部过" -> 流程结束
- 用户未回复前，流程**硬锁在S6出口**

### S6输出要求（硬规则）
S6扫描通过后，**必须将终稿完整输出到可复制代码块**。

## S7 配图（可选·用户触发）

### 触发条件
- S6安全扫描完成后，用户明确说"配图""封面图""头图""需要图"**"1""ok"** 才进入S7
- 用户未提及 -> S6即终稿，流程结束

### S7硬规则
- **不猜测主体**：用户未明确主体 -> 追问，不自动填充
- **不生成图片**：Agent不调用 generate_image
- **模板复用**：提示词必须严格复用 `references/image-standard.md` 模板
- **主体确认流程**：用户说"配图" -> 要求提供四层数字 -> 冲突预判 -> 确认主体 -> 输出提示词

### S7四层数字 N-N-N-N
- **第一层**：1=人物 / 2=物体场景 / 3=动物 / 4=纯文字 / 5=通用
- **第二层**：1=精确还原 / 2=简洁线条
- **第三层**：1=画中画 / 2=直接放置
- **第四层**：1=anthropic / 2=留白守护 / 3=配色融入

纯文字硬规则：第一层`4` -> 第三层固定`1`，第四层默认`3`。

详细提示词模板、配色规范、各变例完整英文模板见 `references/image-standard.md`。
